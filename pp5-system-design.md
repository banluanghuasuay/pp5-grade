# ระบบ ปพ.5 โรงเรียน — เอกสารออกแบบ

> เอกสารสรุปการออกแบบระบบบันทึกผลการเรียน (ปพ.5) สำหรับโรงเรียนขนาดเล็ก
> รองรับทั้งระดับประถมและมัธยม

## 1. ภาพรวมระบบ

ระบบ ปพ.5 ออนไลน์สำหรับโรงเรียนขนาดเล็ก (< 300 คน) รองรับทั้งประถมและมัธยม
มีผู้ใช้ 3 กลุ่ม (Admin, ครู, ผู้ปกครอง) เน้นการบันทึกคะแนน เวลาเรียน และออกรายงาน ปพ.5
ตามรูปแบบกระทรวงศึกษาธิการ

**กลยุทธ์การพัฒนา:** ทำระบบประถมก่อน → ขยายไปมัธยม

## 3. ความแตกต่างประถม vs มัธยม

### ตารางเปรียบเทียบ

| ประเด็น | ประถมศึกษา | มัธยมศึกษา |
|--------|-----------|-----------|
| หน่วยตัดสินผล | รายปี (ทั้งปีการศึกษา) | รายภาคเรียน |
| การลงทะเบียน | ลงครั้งเดียวต่อปี | ลงใหม่ทุกภาคเรียน |
| การคิดคะแนน | ภาค 1 (100) + ภาค 2 (100) ÷ 2 | ภาคละ 100 คะแนน ตัดเกรดเลย |
| ผลการเรียน | 1 เกรด/วิชา/ปี | 1 เกรด/วิชา/ภาค |
| หน่วยกิต | มี (40 ชม. = 1 นก.) | มี (40 ชม. = 1 นก.) |
| ปพ.5 | 1 ฉบับ/วิชา/ปี (มี 2 ภาค) | 1 ฉบับ/วิชา/ภาค |
| กลางภาค | ไม่บังคับ (เลือกได้) | บังคับ |
| GPAX สะสม | ไม่มี | มี (แสดงใน ปพ.1) |

### Flow ประถมศึกษา
```
ภาคเรียนที่ 1 (พ.ค. - ต.ค.)
   ├── บันทึกคะแนน 100 คะแนน
   ├── บันทึกเวลาเรียน
   └── เก็บไว้ (ยังไม่ตัดเกรด)

ภาคเรียนที่ 2 (พ.ย. - มี.ค.)
   ├── บันทึกคะแนน 100 คะแนน
   ├── บันทึกเวลาเรียน
   └── ระบบดึงคะแนนภาค 1 มา + ภาค 2 ÷ 2 → ตัดเกรด

ปลายปีการศึกษา
   └── ออก ปพ.5 รายปี
```

### Flow มัธยมศึกษา
```
ภาคเรียนที่ 1
   ├── ลงทะเบียนวิชา
   ├── บันทึกคะแนน 100 คะแนน
   ├── ตัดเกรดเลย
   └── ออก ปพ.5 ของภาค 1

ภาคเรียนที่ 2
   ├── ลงทะเบียนใหม่ (วิชาคนละตัว)
   ├── บันทึกคะแนน 100 คะแนน
   ├── ตัดเกรดเลย
   └── ออก ปพ.5 ของภาค 2
```

## 5. ประเภทวิชา (Subject Categories)

ทุกวิชาในระบบจัดเป็น 1 ใน 3 ประเภท ตามหลักสูตรแกนกลาง 2551:

| ประเภท | ตัวอย่าง | การประเมิน | นับใน GPA |
|--------|----------|------------|-----------|
| **วิชาพื้นฐาน** (core) | ภาษาไทย, คณิตศาสตร์, วิทย์ฯ | เกรด 0-4 | ✅ |
| **วิชาเพิ่มเติม** (additional) | คอมพิวเตอร์, ภาษาจีน | เกรด 0-4 | ✅ |
| **กิจกรรม** (activity) | ลูกเสือ, ชุมนุม, กิจกรรมเพื่อสังคม | **ผ่าน/ไม่ผ่าน** | ❌ |

### Logic

```
ถ้า subject.grading_mode = 'numeric' (พื้นฐาน + เพิ่มเติม)
    → ใช้ฟิลด์ grade (0, 0.5, 1, ..., 4)
    → ตั้งโครงสร้างคะแนน → กรอกคะแนนเก็บ + ปลายภาค
    → คำนวณเกรดอัตโนมัติ

ถ้า subject.grading_mode = 'pass_fail' (กิจกรรม)
    → ใช้ฟิลด์ pass_fail ('pass' / 'fail')
    → ไม่ต้องตั้งโครงสร้างคะแนน
    → ครูเลือก ผ่าน/ไม่ผ่าน โดยตรง
    → มีปุ่ม "ผ่านทั้งห้อง" ช่วยกรอกเร็ว
```

### หน้าจัดการที่กระทบ
- **แผนการเรียน** — เพิ่มฟิลด์ประเภท · จัดกลุ่มแสดงแยก 3 หมวด · กิจกรรมไม่มีหน่วยกิต
- **โครงสร้างคะแนน** — กิจกรรมข้าม (ไม่ต้องตั้งค่า)
- **บันทึกคะแนน** — กิจกรรมมีตารางแยก (ชื่อ + dropdown ผ่าน/ไม่ผ่าน)
- **ปพ.5** — แยกตาราง 3 ส่วนใน PDF
- **Portal ผู้ปกครอง** — กิจกรรมแสดง "ผ่าน/ไม่ผ่าน" แทนเกรด

### Schema เพิ่ม

```sql
-- subjects เพิ่มฟิลด์
ALTER TABLE subjects ADD COLUMN category VARCHAR(20)
  CHECK (category IN ('core', 'additional', 'activity'));
ALTER TABLE subjects ADD COLUMN grading_mode VARCHAR(20)
  CHECK (grading_mode IN ('numeric', 'pass_fail'));

-- grades รองรับ 2 mode
ALTER TABLE grades ADD COLUMN pass_fail VARCHAR(10)
  CHECK (pass_fail IN ('pass', 'fail'));
-- grades.grade เป็น nullable (ใช้กับ numeric)
-- grades.pass_fail เป็น nullable (ใช้กับ pass_fail)
```

## 6. โครงสร้างคะแนน

### หลักการสำคัญ
**โครงสร้างคะแนน ผูกกับ subject_offering (วิชา × ห้อง × ครู × ภาคเรียน) ไม่ใช่ subject**

หมายความว่า:
- คณิต ม.1/1 ครู A → มีโครงสร้างคะแนน 30-20-30-20
- คณิต ม.1/2 ครู B → มีโครงสร้างคะแนน 40-20-20-20
- คณิต ม.1/3 ครู A → ครู A คนเดิม แต่ตั้งใหม่ก็ได้

ทุกห้องมีอิสระ 100% ในการตั้งสัดส่วน แต่ **เกณฑ์ตัดเกรด** ใช้เหมือนกันทั้งโรงเรียน

### โครงสร้างคะแนน 2 ระดับ

```
หมวด (เก็บก่อนกลาง 30) ─┬─ หัวข้อย่อย 1 (10)
                        ├─ หัวข้อย่อย 2 (10)
                        └─ หัวข้อย่อย 3 (10)
                                       ─────
                                         30 ✓
```

ครูเลือกได้ 2 แบบ:
- กรอกเป็น "หมวด" อย่างเดียว เช่น เก็บก่อนกลาง = 30
- หรือแยกย่อยเป็น 1.1, 1.2, 1.3 (ระบบรวมให้อัตโนมัติ)

### โครงสร้างประถม (default)
```
| คะแนนเก็บ (10 ช่อง) | ปลายภาค | รวม | เกรด |
```
- กลางภาคเป็นตัวเลือก (ติ๊กเปิดได้)

### โครงสร้างมัธยม (default)
```
| ก่อนกลาง (5 ช่อง) | กลางภาค | หลังกลาง (5 ช่อง) | ปลายภาค | รวม | เกรด |
```
- มีเส้นแบ่งซ้ายของแต่ละบล็อก
- แยกสีแต่ละบล็อก (เขียว/เหลือง/ฟ้า/ชมพู)

### ฟีเจอร์เด่นของหน้าตั้งค่า
- ตรวจสอบยอดรวมอัตโนมัติ (✓ = ตรงกับคะแนนเต็มหมวด)
- ลากจัดเรียงได้ (Drag & Drop)
- เพิ่ม/ลบหมวดและหัวข้อย่อย
- เทมเพลตสำเร็จ + บันทึกเทมเพลตของตัวเอง
- ปุ่ม "Copy โครงสร้างจากห้องอื่น" (ช่วยครูที่สอนหลายห้อง)

## 7. หน้าบันทึกคะแนน

### Layout (มัธยม)

```
[ภาคเรียนที่ 1] [ภาคเรียนที่ 2] [สรุปผล/ตัดเกรด]

⚠ ตั้ง "คะแนนเต็ม" บรรทัดแรกก่อนกรอกคะแนน

| ที่ | ชื่อ | ก่อนกลาง 1-5 | กลาง | หลังกลาง 6-10 | ปลาย | รวม | เกรด |
|----|------|--------------|------|----------------|------|-----|------|
| -  | คะแนนเต็ม | 10 10 10 0 0 | 20 | 10 10 10 0 0 | 20 | 100 | - |
| 1  | สมชาย | 9 8 9 - - | 18 | 9 8 9 - - | 18 | 88 | 4 |
| 2  | สมหญิง | 7 8 7 - - | 14 | 7 8 7 - - | 14 | 72 | 3 |

### ฟีเจอร์เด่น
- กรอกเร็วเหมือน Excel (Tab/Enter ไปช่องถัดไป, ลูกศร 4 ทิศ)
- บันทึกอัตโนมัติ (auto-save)
- แถวคะแนนเต็มอยู่บนสุดในตารางเดียวกัน
- ช่องที่ไม่ใช้ ใส่ 0 → สีจาง บอกว่า "ไม่ได้ใช้"
- คอลัมน์เกรดอยู่ขวาสุดในตารางเดียวกัน
- สีเกรด: เขียว (4) / เหลือง (3) / ส้ม (2) / ชมพู (1, 0)
- เตือนถ้ากรอกคะแนนเกินคะแนนเต็ม (สีแดง)
- ปุ่ม Excel + ล็อกคะแนน อยู่มุมขวาล่าง
- ปุ่ม "ตั้งค่าหัวข้อ" สำหรับปรับสัดส่วน

### Layout (ประถม)
- ไม่มีบล็อกกลางภาค (default)
- แท็บ "ภาคเรียนที่ 2" จะมีคอลัมน์เพิ่ม "คะแนนภาค 1 (อ้างอิง)" เป็น read-only

### Layout (มัธยม) - ตายตัว
- ก่อนกลางภาค: **5 ช่อง** (ตายตัว)
- กลางภาค: 1 ช่อง
- หลังกลางภาค: **5 ช่อง** (ตายตัว)
- ปลายภาค: 1 ช่อง
- ครูใส่ 0 ในช่องที่ไม่ใช้ → ระบบทำสีจาง

### Lifecycle โครงสร้างคะแนน (2 สถานะ)

| สถานะ | คำอธิบาย | แก้ไขได้? |

| 2. ตั้งแล้ว | แก้ไขทุกอย่างได้ตลอด | ✅ แก้ได้อิสระ |

- ปุ่ม "ตั้งโครงสร้าง" สำหรับงานซับซ้อน (เปิด/ปิดกลางภาค, copy เทมเพลต)

## 8. หน่วยกิต (ใช้เหมือนกันทั้ง 2 ระดับ)

- **40 ชม. = 1 นก.** (ตามหลักสูตรแกนกลาง 2551)
- คำนวณอัตโนมัติ: หน่วยกิต = (ชม./สัปดาห์ × สัปดาห์) ÷ 40
- ตัวอย่าง: เรียน 5 ชม./สัปดาห์ × 20 สัปดาห์ = 100 ชม. → 2.5 นก./ภาค

### GPA Formula (ถ่วงน้ำหนักด้วยหน่วยกิต)

```
GPA = Σ(เกรด × หน่วยกิต) ÷ Σ(หน่วยกิต)
```

### ตัวอย่าง (ประถม ป.3 ภาค 1)

| วิชา | นก. | เกรด | เกรด × นก. |
|------|-----|------|----------|
| ภาษาไทย | 2.5 | 4 | 10.0 |
| คณิตศาสตร์ | 2.5 | 3.5 | 8.75 |
| วิทยาศาสตร์ | 2.0 | 3 | 6.0 |
| สังคม | 2.0 | 4 | 8.0 |
| ภาษาอังกฤษ | 1.0 | 3.5 | 3.5 |
| สุขศึกษา | 1.0 | 4 | 4.0 |
| ศิลปะ | 1.0 | 4 | 4.0 |
| การงาน | 1.0 | 3.5 | 3.5 |
| **รวม** | **13.0** | | **47.75** |

**GPA = 47.75 ÷ 13.0 = 3.67**

## 9. การลงทะเบียนและการย้ายเข้า-ออก

**หลักการ: ระบบ ปพ.5 เน้นที่คะแนน · ทะเบียนนักเรียนใช้ระบบอื่น**

> นักเรียนเพิ่ม/ลบได้ตลอด · ไม่ track ประวัติย้ายเข้า-ออก
> ระบบทะเบียนนักเรียนของโรงเรียน (ที่มีอยู่แล้ว) จัดการประวัติแยกต่างหาก

### Auto-enrollment
- นักเรียนในห้องไหน → ลงทะเบียนวิชาในแผนของห้องนั้น**อัตโนมัติ ไม่มี popup ถาม**
- ใช้กับวิชาบังคับ (พื้นฐาน) เป็น default
- วิชาเลือก/เพิ่มเติม → ปิด auto-enroll ให้เลือกเอง

### กรณีการย้าย 3 แบบ

**1. ย้ายเข้าใหม่ (กลางภาค)**
- เพิ่มข้อมูลนักเรียน + ลงทะเบียนเข้าห้อง
- Auto-enroll วิชาทั้งหมดของห้องนั้น**ทันที ไม่ถาม**
- ครูเห็นในตารางบันทึกคะแนน **เหมือนนักเรียนคนอื่น ไม่มีป้ายพิเศษ**
- ครูเจอช่องว่าง → ใส่คะแนนตามความเหมาะสม

**2. ย้ายห้อง (ภายในโรงเรียน)**
- ลบ enrollment เก่า + สร้างใหม่ในห้องใหม่
- **คะแนนเดิมไม่ย้ายตาม** (ต้อง re-record ในห้องใหม่)
- ครูเก่าและครูใหม่จะเห็นคนละ scope

**3. ย้ายออก / จบการศึกษา**
- ลบนักเรียนจากระบบ (CASCADE — scores/grades/attendance หายตาม)
- UI ต้องเตือนก่อนลบ
- ถ้าต้องการ ปพ.5 ย้อนหลัง → ใช้ระบบทะเบียนหลักของโรงเรียน

### Schema จริง

- `students` — ข้อมูลพื้นฐาน (ชื่อ, รหัส, เพศ, วันเกิด)
- `enrollments` — link student ↔ classroom · เพิ่ม/ลบได้ตลอด
- **ไม่มี** `status` / `enrollment_date` / `notes` / `transferred_at` — ตามที่ตัดออก

## 12. Tech Stack + Architecture

### Architecture: แยก 2 เว็บ + Database เดียว

```
parent.school.com           admin.school.com
─────────────────           ─────────────────
Portal ผู้ปกครอง            หลังบ้าน
(read-only)                 (Admin + ครู)
Mobile-first                Responsive (Desktop + Mobile)
       │                            │
       └──────────┬─────────────────┘
                  ↓
          ┌───────────────┐
          │   Supabase    │
          │ PostgreSQL+Auth│
          └───────────────┘
```

**เหตุผล:**
- ✅ แยก concerns: Portal เบา, หลังบ้านครบฟีเจอร์
- ✅ ปลอดภัยขึ้น: Portal ไม่มี route admin
- ✅ ครูบันทึกคะแนนบนมือถือได้ (หลังบ้าน responsive)
- ✅ Database เดียว → ไม่ต้อง sync

### Tech Stack

```
Frontend:  Next.js 15 (App Router) × 2 apps
UI:        TailwindCSS + shadcn/ui
DB+Auth:   Supabase (PostgreSQL + Row-Level Security)
Deploy:    Vercel × 2 projects + Supabase
PDF:       Puppeteer (HTML → PDF, ฟอนต์ไทยดีกว่า)
Excel:     SheetJS (xlsx)
Charts:    recharts
Monorepo:  pnpm workspaces หรือ Turborepo
```

### Codebase Structure (Monorepo)

```
pp5-system/
├── apps/
│   ├── admin/                  # admin.school.com
│   └── parent/                 # parent.school.com
├── packages/
│   ├── database/               # Supabase schema + types
│   ├── ui/                     # Shared components
│   └── lib/                    # Utility (grading, pdf, ฯลฯ)
└── package.json
```

**ทำไม Monorepo:** แชร์ component, types, business logic ระหว่าง 2 เว็บ — ไม่ต้องเขียนซ้ำ

### Mobile Strategy
- **Portal ผู้ปกครอง** → Mobile-first design (เว็บ responsive)
- **หลังบ้าน** → Desktop-first + Mobile responsive (ครูใช้ในห้องเรียน)
- **ไม่มี Mobile App ในตอนนี้** (Phase ถัดไปอาจทำ PWA หรือ React Native)

### ค่าใช้จ่าย
- เริ่มต้น: **0 บาท/เดือน** (Vercel free × 2 + Supabase free)
- ถ้าโตขึ้น: **~$25/เดือน** (Supabase Pro) ≈ 900 บาท

### เหตุผลที่เลือก Supabase
- Auth, DB, Storage พร้อมใช้
- Row-Level Security จัดการสิทธิ์ครู/admin/ผู้ปกครองได้สะดวก
- ไม่ต้องเขียน backend เอง
- Deploy ง่าย ดูแลง่าย

## 14. ข้อควรระวังเฉพาะระบบ ปพ.5

- **เกณฑ์เกรด 8 ระดับ** (0, 1, 1.5, 2, 2.5, 3, 3.5, 4) — ตั้งค่าได้
- **เวลาเรียน 80%** — ต้องคำนวณและแจ้งเตือน "มส." (เวลาเรียนไม่พอ)
- **คะแนน 0-100** — แต่ละวิชากำหนดสัดส่วนได้เอง
- **มาตรฐาน/ตัวชี้วัด** — ปพ.5 จริงต้องระบุ ควรมีฟิลด์รองรับ
- **ลายเซ็นดิจิทัล** — ผู้บริหาร/ครูประจำชั้น อัปโหลดลายเซ็นไว้ใช้ใน PDF
- **สำรองข้อมูล** — ต้องมี backup อัตโนมัติ
- **คะแนนเกินคะแนนเต็ม** → แสดงสีแดง (visual feedback) แต่ไม่ block การบันทึก

### ผลการเรียนพิเศษที่ต้องรองรับ
- ร (รอผล)
- มส (เวลาเรียนไม่พอ)
- ผ/มผ (ผ่าน/ไม่ผ่าน) — สำหรับกิจกรรมพัฒนาผู้เรียน
- การสอบซ่อม

## 19. การพิมพ์ ปพ.5 PDF

ปพ.5 = เอกสารทางการที่ต้องส่งเข้ากระทรวงและเก็บเป็นหลักฐาน

### 2 รูปแบบการพิมพ์

#### แบบ A: รายวิชา (1 ปพ.5 = 1 วิชา)
- **เหมาะกับ:** มัธยม + ประถมขนาดใหญ่
- **5 หน้า:** ข้อมูลวิชา → คะแนน 1 วิชา → เวลาเรียนวิชานั้น → ประเมินวิชานั้น → สรุป+ลงนาม
- **ลงนาม:** ครูผู้สอน · หัวหน้ากลุ่มสาระ · ผอ.
- **1 ห้อง 8 วิชา → 8 เล่ม**

#### แบบ B: รวมห้อง (1 ปพ.5 = ทุกวิชาของห้อง)
- **เหมาะกับ:** ประถมขนาดเล็ก
- **โครงสร้าง:**
  - หน้าปก: ข้อมูลห้อง + ครูประจำชั้น + **สารบัญรายวิชา** (รหัส/ชื่อ/ครูผู้สอน/หน้า)
  - หน้าคะแนนแต่ละวิชา (วิชาละ 1 หน้า) — มี header บอกครูผู้สอน + ลงนามครูผู้สอน + ครูประจำชั้น
  - หน้าบันทึกเวลาเรียน — ครูประจำชั้นบันทึก
  - หน้าประเมินคุณลักษณะ/อ่านคิดเขียน/สมรรถนะ — ครูประจำชั้นประเมิน
  - หน้าสรุป + ลงนาม
- **ลงนามรวม:** ครูประจำชั้น · ครูวิชาการ · ผอ.
- **1 ห้อง → 1 เล่ม (~12 หน้า)**

> **สำคัญ:** ปพ.5 รวมห้องมีครูผู้สอนหลายคน (ตามวิชา) แม้ครูประจำชั้นจะรับผิดชอบเล่มโดยรวม

ครูเลือกได้ทุกครั้งที่พิมพ์ — โรงเรียนเดียวกันอาจใช้ทั้ง 2 แบบในบริบทต่างกัน

### โครงสร้าง 5 หน้า (แบบ A - รายวิชา)

| หน้า | เนื้อหา | ขนาด |
|------|--------|------|
| 1 | ข้อมูลทั่วไป (รหัสวิชา ครู ปีการศึกษา) + สรุปโครงสร้างคะแนน + เกณฑ์เกรด | A4 แนวตั้ง |
| 2 | ตารางคะแนนรายคน (เก็บ + ปลายภาค + รวม + เกรด) | A4 แนวนอน |
| 3 | บันทึกเวลาเรียน (มา/ขาด/ลา/ป่วย/%) | A4 แนวตั้ง |
| 4 | ประเมิน 3 หมวด: คุณลักษณะ + อ่านคิดเขียน + สมรรถนะ | A4 แนวนอน |
| 5 | สรุปสถิติการกระจายเกรด + ลงนาม 3 คน | A4 แนวตั้ง |

### Features หน้าพิมพ์ (UI สำหรับครู)

1. **เลือกรูปแบบ** — A (รายวิชา) / B (รวมห้อง)
2. **เลือกข้อมูล** — ห้อง / วิชา (เฉพาะแบบ A) / ภาคเรียน
3. **ตัวเลือก:**
   - ใส่/ไม่ใส่โลโก้โรงเรียน
   - ใส่/ไม่ใส่หมายเลขหน้า
   - ลายน้ำ "DRAFT" (ฉบับร่าง)
   - พิมพ์เฉพาะที่ไม่ผ่าน
4. **Output:** PDF (default) / Excel / Word
5. **Live preview** ก่อนพิมพ์

### Tech Implementation

```

- ฟอนต์: Sarabun New หรือ TH Sarabun PSK (มาตรฐานราชการ)

## 21. ภาคผนวก: การประเมินตามหลักสูตร

รวมอยู่ในหน้าเดียว แบ่งเป็น 4 แท็บ — ใช้เกณฑ์ 4 ระดับเหมือนกัน:
- **3** = ดีเยี่ยม (พื้นเขียว)
- **2** = ดี (พื้นฟ้า)
- **1** = ผ่าน (พื้นเหลือง)
- **0** = ไม่ผ่าน (พื้นแดง)

### หลักการประเมิน (สำคัญ)
ครูเลือกระดับ 0-3 ของ "คุณลักษณะแต่ละข้อ" โดยตรง (ไม่กรอกระดับตัวชี้วัดย่อย) — เพื่อความเรียบง่ายในการใช้งาน · ระบบคำนวณสรุปรวมตามสูตร สพฐ. ให้อัตโนมัติ

### แท็บ 1: ตั้งค่าคุณลักษณะฯ (แก้ไขได้)

ค่าเริ่มต้น 8 ข้อตาม สพฐ. — เพิ่ม/ลบ/แก้ไข/ลากจัดเรียงได้
1. รักชาติ ศาสน์ กษัตริย์
2. ซื่อสัตย์สุจริต
3. มีวินัย
4. ใฝ่เรียนรู้
5. อยู่อย่างพอเพียง
6. มุ่งมั่นในการทำงาน
7. รักความเป็นไทย
8. มีจิตสาธารณะ

**ป้ายที่มา:** 🟣 สพฐ. / 🔵 โรงเรียน · ปุ่ม "โหลดค่าเริ่มต้น สพฐ." กรณีต้องการรีเซ็ต

### แท็บ 2: ประเมินคุณลักษณะฯ

- ตาราง: นักเรียน × หัวข้อที่ตั้งไว้ + สรุป
- Dropdown ต่อช่อง — พื้นสีตามค่า

**สูตรสรุป 8 ข้อ (ตาม สพฐ. 2551):**
```
1. มีข้อใดได้ "ไม่ผ่าน" (0)         → สรุป = 0 ไม่ผ่าน
2. ดีเยี่ยม (3) ≥ 5 ข้อ              → สรุป = 3 ดีเยี่ยม
3. ดีเยี่ยม + ดี รวม ≥ 5 ข้อ         → สรุป = 2 ดี
4. ที่เหลือ                          → สรุป = 1 ผ่าน
```

### แท็บ 3: ประเมินอ่าน คิดวิเคราะห์ฯ (ตายตัว 3 ด้าน)
1. การอ่าน
2. การคิดวิเคราะห์
3. การเขียน

**สูตรสรุป 3 ด้าน (ตาม สพฐ. 2551):**
```
1. มีด้านใดได้ "ไม่ผ่าน" (0)         → สรุป = 0 ไม่ผ่าน
2. ดีเยี่ยม (3) ทั้ง 3 ด้าน          → สรุป = 3 ดีเยี่ยม
3. ดีเยี่ยม + ดี ทั้ง 3 ด้าน (มี 3 ≥ 1) → สรุป = 2 ดี
4. ที่เหลือ                          → สรุป = 1 ผ่าน
```

### แท็บ 4: ประเมินสมรรถนะสำคัญ (ตายตัว 5 ด้าน)
1. ความสามารถในการสื่อสาร
2. ความสามารถในการคิด
3. ความสามารถในการแก้ปัญหา
4. ความสามารถในการใช้ทักษะชีวิต
5. ความสามารถในการใช้เทคโนโลยี

**สูตรสรุป 5 ด้าน (ตาม สพฐ. 2551):**
```
1. มีด้านใดได้ "ไม่ผ่าน" (0)         → สรุป = 0 ไม่ผ่าน
2. ดีเยี่ยม (3) ≥ 3 ด้าน              → สรุป = 3 ดีเยี่ยม
3. ดีเยี่ยม + ดี รวม ≥ 3 ด้าน         → สรุป = 2 ดี
4. ที่เหลือ                          → สรุป = 1 ผ่าน
```

### Pseudo-code

```typescript
function summarize(scores: number[], threshold: number): number {
  if (scores.includes(0)) return 0;
  const count3 = scores.filter(s => s === 3).length;
  const count2 = scores.filter(s => s === 2).length;
  if (count3 >= threshold) return 3;
  if (count3 + count2 >= threshold) return 2;
  return 1;
}

// เรียกใช้:
summarize(characteristics8, 5);   // คุณลักษณะ 8 ข้อ
summarize(reading3, 3);            // อ่านคิดเขียน 3 ด้าน
summarize(competencies5, 3);       // สมรรถนะ 5 ด้าน
```
