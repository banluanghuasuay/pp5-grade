# การออกเวอร์ชันใหม่ (สำหรับผู้พัฒนา)

ระบบมีแถบแจ้งเตือน "มีเวอร์ชันใหม่" ในแอป admin ที่เทียบเวอร์ชันของแต่ละ
โรงเรียนกับ `apps/admin/version.json` บน branch `main` ของ repo หลักนี้

## เวลาจะแจ้งทุกโรงเรียนว่ามีอัปเดต

1. แก้ `apps/admin/version.json`
   - เพิ่มเลข `version` (เช่น `1.0.0` → `1.1.0`)
   - ใส่ `notes` สั้น ๆ ว่าอัปเดตอะไร (จะแสดงในแถบ)
2. `commit` + `push` เข้า `main`
3. โรงเรียนที่ใช้ระบบจะเห็นแถบ "มีเวอร์ชันใหม่" ภายใน ~1 ชั่วโมง
   (cache) แต่ละโรงเรียนกด **Sync fork** + Redeploy เองเพื่ออัปเดต

## หลักการเลขเวอร์ชัน (semver: major.minor.patch)

| เปลี่ยน | ตัวอย่าง | ใช้เมื่อ |
|--------|---------|---------|
| patch  | 1.0.0 → 1.0.1 | แก้บั๊กเล็กน้อย |
| minor  | 1.0.0 → 1.1.0 | เพิ่มฟีเจอร์ |
| major  | 1.0.0 → 2.0.0 | เปลี่ยนใหญ่ / ต้องรัน SQL เพิ่ม |

## ข้อจำกัดที่ต้องรู้

- โรงเรียนที่ deploy ไว้ **ก่อน** มีฟีเจอร์แถบแจ้งเตือนนี้ จะยัง**ไม่เห็นแถบ**
  จนกว่าจะ Sync fork ครั้งแรก (เพื่อรับโค้ดแถบ) — ครั้งแรกสุดต้องแจ้งเขาเอง
- การเทียบใช้ไฟล์ `apps/admin/version.json` บน `main` เท่านั้น —
  อย่าลืม push ขึ้น `main` ไม่งั้นโรงเรียนจะไม่เห็น
- ถ้าอัปเดตมี migration SQL → ต้องแจ้งโรงเรียนแยกต่างหากให้รัน SQL ด้วย
  (แถบบอกแค่ว่า "มีเวอร์ชันใหม่" ไม่ได้รัน SQL ให้)
